Spotify Top 100 2017 Analysis

November 9, 2018

Background

This project was a parallel training session during my university studies, to practice for my practical exam in the Intro Analytics Class. It was also my first time trying and working with Kaggle and a “new” non-test Dataset (like cars or diamonds).

Therefore this project was quite old and did not use all of the tricks and code practices I know at this point. I revisited it and rewrote most of the code and plots.

You can find my code on GitHub here.

Idea and Hypothesis

The dataset contained the Top 100 Songs on Spotify from 2017. Additionally it gives numeric values for specific feelings or moods for the song. This includes things like the valence/happiness, danceability or energy. There are also meta information about the song like its key, mode and BPM.

I wanted to find out if there are some aspects out of this dataset, that correlate with the position of the song in the Top 100. For example, do happier songs perform better? Is a certain speed better? And how do the attributes correlate?

Method and analysis

My first step was to explore the given values over the position of the tracks. The plot matrix can be seen below.

Here we can see that the values span different numeric ranges. The computed mood values mentioned range from 0 to 1 whereas the tempo ranges from the usual 80 to 200 BPM. Using a y ~ x formula we can also see a first correlation, indicating if a higher value could result in a higher or lower position.

The significance of these correlation would be the next step. Using a linear regression, it would become clear that no attribute could pass a 5% rate. The closest ones however would be the Key and the Valence. Both are plotted below.

As you can see, a key further away from C (B being the furthest) results in a lower position. This is only close to being significant however (p-value 0.026). Additionally a happier song results in a higher position in the Top 100

Next up was the question how attributes themselves correlate. To answer this I created a correlation Matrix with the +/- correlation. As you can see below, the biggest correlation can be found in the Loudness x Energy (+0.71) and Loudness x Speechiness (-0.44). Other notables include Valence and danceability (+0.42) or Tempo and Danceability (-0.36).

The latter brings up the question, which tempo is the most danceable, according to this dataset. A plot visualizing that can be found below. As you can see, around 120 BPM is the best for danceable songs

Next Steps

This analysis is relatively complete. There could always be other combinations to look for, but the analysis does not signify any significant correlation and insight. Other questions answered include if one key is particularly happy, which does not seem to be the case. However, this exercise was interesting regarding its background and available information.
